# redrob-v4 · Team Antigravity

**Intelligent candidate ranking for a Retrieval / Recommendation Systems Engineer role.**

Processes 100,000 raw candidate resumes and surfaces the top 100 best-fit matches using a three-signal ensemble — deterministic feature rules, dense semantic embeddings, and BM25 keyword matching.

---

## Quickstart

Place `candidates.jsonl` in the parent directory of this repo, then:

```bash
# Clone and pull LFS artifacts
git lfs install && git lfs pull

# Install dependencies
python -m venv .venv && source .venv/bin/activate
pip install sentence-transformers rank_bm25 numpy

# Run (auto-parses candidates on first run ~2 min, then ranks ~1 min)
python rank.py --candidates ../candidates.jsonl --out ../submission.csv
```

Expected directory layout:

```
parent/
├── redrob-v4/          ← this repo
└── candidates.jsonl    ← raw dataset (not tracked in repo)
```

## How It Works

```
candidates.jsonl  →  01_parse  →  02_features  →  03_bm25  →  04_embeddings  →  rank.py  →  submission.csv
```

| Stage | Script | Output | Runtime |
|---|---|---|---|
| 1 | `01_parse_candidates.py` | `candidates_parsed.pkl` | ~2 min |
| 2 | `02_extract_features.py` | `feature_matrix.pkl` | ~2 min |
| 3 | `03_build_bm25.py` | `bm25_scores.pkl` | ~3 min |
| 4 | `04_build_embeddings.py` | `embeddings.npy` | ~55 min (CPU) |
| 5 | `rank.py` | `submission.csv` | ~1 min |

**Stages 2–4 are pre-computed and included via Git LFS.** Only stage 1 runs on first use.

To regenerate everything from scratch (e.g., new candidates file):
```bash
python rank.py --candidates ../candidates.jsonl --out ../submission.csv
```
This automatically re-runs all 4 precompute stages before ranking.

## Scoring

```
Final Score = 0.50 × Feature Score + 0.30 × Embedding Score + 0.20 × BM25 Score
```

### Feature Score (50%)

| Signal | Weight | What it measures |
|---|---|---|
| Production Retrieval | 30% | Built & shipped vector DBs / search systems (Pinecone, FAISS, Qdrant…) |
| Product Company | 20% | Career fraction at product companies vs. consulting firms |
| Eval Depth | 15% | NDCG, MRR, A/B testing, precision@k mentioned |
| Domain Fit | 15% | NLP/IR vs. CV/Speech domain alignment |
| Experience Range | 10% | Sweet spot: 5–9 years = 1.0 |
| Location | 5% | Pune/Noida preferred; willing-to-relocate treated as local |
| Engagement | 5% | Notice period, recency, recruiter response rate |

### Hard Disqualifiers → score becomes 0.0

`is_honeypot` · `is_consulting_only` · `is_pure_research` · `is_llm_wrapper_only` · `is_not_coding_ic` · `is_wrong_domain`

### Soft Penalizers (multiplicative)

`is_title_chaser` ×0.75 · `is_framework_only` ×0.80 · `is_closed_source` ×0.85 · low recruiter response ×0.85 · notice period ×0.55–1.0 · recency ×0.70–1.0

## Artifacts

Pre-computed artifacts are tracked via **Git LFS**. Run `git lfs pull` after cloning.

| File | Size | Description |
|---|---|---|
| `embeddings.npy` | 147 MB | 384-dim candidate vectors (all-MiniLM-L6-v2) |
| `jd_embedding.npy` | 4 KB | 384-dim JD vector |
| `feature_matrix.pkl` | 17 MB | Feature dicts for all 100K candidates |
| `bm25_scores.pkl` | 2.3 MB | BM25 score per candidate |
| `honeypots_flagged.json` | 1.2 MB | Flagged honeypot IDs |

`candidates_parsed.pkl` (~398 MB) is not tracked — too large for LFS free tier. It is auto-generated by `rank.py` on first run.
